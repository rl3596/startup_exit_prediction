# 特征矩阵说明文档

**文件**: `feature_matrix.csv`
**行数**: 6,704 家公司（美国 AI 初创企业，融资总额 >= 100万美元）
**列数**: 37（1 个 ID + 1 个目标变量 + 35 个特征）

## 标识符与目标变量

| 列名 | 类型 | 说明 |
|------|------|------|
| `company_uuid` | ID | 公司唯一标识符，不作为特征使用。 |
| `is_success` | 目标变量 (0/1/null) | 机器学习目标变量。1 = 成功退出（IPO、被收购、独角兽估值，或达到按年龄分层的融资门槛）；0 = 未成功；Null = 数据不足，无法标注。 |

## 网络特征（图派生特征）

这些特征基于**投资者-公司二部图**（22,807 个节点，47,512 条边）和**联合投资图**（15,766 个投资者节点，173,986 条边——若两个投资者参与了同一轮融资则连边）计算得出。

### 公司级中心性特征

| 列名 | 类型 | 说明 |
|------|------|------|
| `company_degree` | int | 投资该公司的不同投资者数量。度数越高 = 投资者连接越多，直接衡量公司在投资网络中的"受欢迎程度"。 |
| `company_degree_centrality` | float | 公司在二部图中的标准化度中心性。与 `company_degree` 信息相同，但按可能连接总数归一化至 [0, 1] 区间。 |
| `company_pagerank` | float | 公司在投资者-公司二部图中的 PageRank 得分（阻尼系数 = 0.85）。衡量递归重要性：当一家公司的投资者本身也投资了许多重要公司时，该公司得分更高。捕捉超越投资者数量的"声望"指标。 |
| `company_betweenness` | float | 近似中介中心性（k=500 随机采样）。衡量该公司位于其他节点对之间最短路径上的频率。高中介中心性 = 该公司连接了原本分离的投资者社区，起到桥梁作用。 |

### 投资者级汇总特征（按公司聚合）

对每家公司，我们查看其所有投资者并汇总各投资者的网络指标。

| 列名 | 类型 | 说明 |
|------|------|------|
| `avg_investor_degree` | float | 公司所有投资者在二部图中度数的均值。每个投资者的度数 = 其投资的公司数量。数值高意味着该公司的投资者平均在大量被投公司中都很活跃。**模型中最重要的预测特征。** |
| `max_investor_degree` | float | 公司所有投资者中度数的最大值。捕捉公司是否拥有至少一个拥有超大投资组合的"超级连接者"风投。 |
| `avg_investor_pagerank` | float | 公司所有投资者 PageRank 的均值。衡量投资者团队的平均"声望"——投资者自身是否与重要公司有良好连接？ |
| `max_investor_pagerank` | float | 公司所有投资者中 PageRank 的最大值。捕捉支持该公司的最具声望/影响力的单一投资者。 |
| `avg_investor_betweenness` | float | 公司所有投资者中介中心性的均值。数值高 = 投资者倾向于连接投资网络中不同社区，可能为公司带来多元化资源。 |

### 联合投资 / 辛迪加特征

| 列名 | 类型 | 说明 |
|------|------|------|
| `investor_clustering_coeff` | float | 公司所有投资者在联合投资图中的平均局部聚类系数（按共同参与轮次加权）。衡量公司的投资者之间总体上联合投资的频率。数值高 = 投资者组成了经常合作的紧密辛迪加。 |
| `investor_coinv_density` | float | 公司所有投资者在联合投资图中诱导子图的边密度。对于有 N 个投资者的公司，这是所有可能的投资者配对中实际有过联合投资的比例。1.0 = 每对投资者都曾联合投资；0.0 = 没有任何配对联合投资过。衡量该公司投资者群体特有的辛迪加凝聚力。 |
| `num_lead_investors` | int | 公司所有融资轮次中不同领投投资者的数量（来源于 `round_investors.is_lead`）。领投投资者通常负责谈判条款并进行更深入的尽职调查。 |

## 表格特征（非图特征）

### 公司财务特征

| 列名 | 类型 | 说明 |
|------|------|------|
| `funding_total_usd` | float | 融资总额（美元）。**已排除在模型训练之外**，因为 `is_success` 的定义部分基于融资门槛——纳入该特征会导致数据泄漏。保留在 CSV 中仅供参考。 |
| `log_funding` | float | `log(1 + funding_total_usd)`。**已排除在模型训练之外**，原因同上。 |
| `num_funding_rounds` | int | 公司完成的融资轮次总数。**已排除在模型训练之外**——与融资总额高度相关，而融资总额用于定义目标变量。 |
| `employees_ordinal` | int (1-9) | 公司规模序数编码：1 = 1-10人，2 = 11-50人，3 = 51-100人，4 = 101-250人，5 = 251-500人，6 = 501-1000人，7 = 1001-5000人，8 = 5001-10000人，9 = 10001人以上。未披露时为 Null（约0.5%缺失）。 |
| `company_age_months` | float | 截至2025年12月31日的公司年龄（月），根据 `founded_on` 计算。**已排除在模型训练之外**——直接用于目标变量的年龄分层计算。 |

### 最近一轮融资类型（独热编码）

最近一轮融资的类型，进行独热编码。保留前8个主要类别；稀有类型归入 `fund_type_other`。

| 列名 | 类型 | 说明 |
|------|------|------|
| `fund_type_pre_seed` | 0/1 | 最近一轮为种子前轮。非常早期阶段，通常 < 200万美元。 |
| `fund_type_seed` | 0/1 | 最近一轮为种子轮。早期阶段，通常 100-500万美元。 |
| `fund_type_series_a` | 0/1 | 最近一轮为 A 轮。首轮机构融资，通常 500-2000万美元。 |
| `fund_type_series_b` | 0/1 | 最近一轮为 B 轮。成长阶段，通常 1500-5000万美元。 |
| `fund_type_series_c` | 0/1 | 最近一轮为 C 轮。晚期成长阶段，通常 3000万-1亿美元以上。 |
| `fund_type_series_unknown` | 0/1 | 最近一轮为未标明轮次编号的系列融资。 |
| `fund_type_grant` | 0/1 | 最近一轮为政府拨款（非稀释性资金）。 |
| `fund_type_non_equity_assistance` | 0/1 | 最近一轮为非股权资助（如加速器项目）。 |
| `fund_type_other` | 0/1 | 最近一轮为前8类之外的其他类型（如可转换票据、债务融资、二级市场交易等）。 |

### 创始人特征

| 列名 | 类型 | 说明 |
|------|------|------|
| `num_founders` | int | Crunchbase 中列出的公司创始人数量。 |
| `founder_has_phd` | 0/1 | 若任一创始人持有博士学位则为1。作为深度技术专长的代理指标。基于 `education.degree_type` 的模式匹配（匹配 "PhD"、"Ph.D"、"Doctor"）。约44%的创始人教育数据缺失，因此该值为下界估计。 |
| `founder_has_mba` | 0/1 | 若任一创始人持有 MBA 学位则为1。作为商业/管理培训的代理指标。同样存在教育数据缺失的问题。 |
| `max_founder_prior_jobs` | int | 所有创始人中历史工作数量的最大值（来源于 `jobs` 表）。作为最有经验创始人的代理指标——连续创业者或行业资深人士通常拥有更长的工作履历。 |
| `avg_founder_prior_jobs` | float | 所有创始人历史工作数量的均值。衡量团队整体经验水平。 |

### 团队构成特征

| 列名 | 类型 | 说明 |
|------|------|------|
| `team_size` | int | 通过 `company_team` 关联到该公司的总人数（高管、董事会成员、顾问）。不包含全部员工——仅包含在 Crunchbase 上有档案的人员。 |
| `c_suite_count` | int | C级高管人数（CEO、CTO、CFO、COO 等）。 |
| `board_count` | int | 董事会成员人数。可以反映公司治理成熟度或投资者参与程度。 |
| `advisor_count` | int | 正式顾问人数。在基线逻辑回归模型中，顾问数量过多是负面信号（权重 -0.27）。 |

## 数据泄漏说明

以下三个特征**已排除在模型训练之外**，但保留在 CSV 中：

| 排除的特征 | 原因 |
|-----------|------|
| `funding_total_usd` / `log_funding` | `is_success` 的定义基于按年龄分层的融资门槛。纳入融资金额会形成循环论证。 |
| `num_funding_rounds` | 与融资总额高度相关。 |
| `company_age_months` | 直接用于 `is_success` 的年龄分层计算。 |

当纳入这些泄漏特征时，模型 ROC-AUC 达到 0.997（99.7%），证实了数据泄漏的存在。排除后，真实 ROC-AUC 为 0.774——主要由网络特征驱动。

## 缺失值

| 列名 | 缺失数 | 缺失比例 |
|------|-------:|--------:|
| `employees_ordinal` | 32 | 0.5% |
| `is_success` | 111 | 1.7% |

其余所有列均无缺失值。111 家 `is_success` 为 null 的公司在模型训练时被排除。
